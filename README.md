# معتز العلقمي

نسخة مرتبة ومراجعة من مشروع المدونة، مهيأة للنشر على **Vercel** مع **Prisma** و**PostgreSQL** و**Neon**.

## لماذا هذه النسخة أفضل؟
- إصلاح أخطاء بنيوية في السكربت الأصلي.
- إزالة نقاط كسر كانت تمنع البناء على Vercel.
- جعل الجلسات في Auth.js بنمط `jwt` لتخفيف مشاكل Edge/middleware.
- إصلاح روابط ومسارات الملفات.
- إصلاح regex في `slugifyArabic`.
- إضافة صفحات أساسية كانت مفقودة.
- إضافة `tailwind.config.ts`.
- تضمين أنواع NextAuth بشكل صحيح.

## التشغيل محليًا
```bash
cp .env.example .env
npm install
npx prisma db push
npm run db:seed
npm run dev
```

## النشر على Vercel
1. ارفع المشروع إلى GitHub.
2. أنشئ قاعدة PostgreSQL على Neon.
3. أضف متغيرات البيئة من `.env.example` في لوحة Vercel.
4. اجعل:
   - `DATABASE_URL` من Neon
   - `AUTH_SECRET` قيمة قوية
   - `AUTH_TRUST_HOST=true`
   - مزود OAuth واحدًا على الأقل
5. نفذ:
```bash
npm run build
```

## هل تحتاج المدونة قاعدة بيانات؟
نعم، هذه النسخة تحتاج قاعدة بيانات لأنها تعتمد على:
- المقالات
- المستخدمين والصلاحيات
- التعليقات
- الاشتراكات البريدية
- الرسائل
- الإعدادات
- الصفحات

## هل Neon مناسب؟
نعم، **Neon مناسب جدًا** لهذه المدونة على Vercel لأنه:
- PostgreSQL managed
- سريع ومناسب لـ Prisma
- سهل الربط مع Vercel
- جيد للتوسع لاحقًا
